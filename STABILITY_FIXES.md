# 🚀 Серверні оптимізації для стабільної роботи

## Проблема
Сервер періодично перевантажував себе через утечки пам'яті і некоректну конфігурацію.

## Внесені фіксації ✅

### 1. **PM2 Конфігурація** (`ecosystem.config.cjs`)
- **Зменшено** `max_memory_restart` з 1G на **512M**
  - Процес тепер перезавтажиться раніше, до критичного стану
- **Додано** параметри стабільності:
  - `max_restarts: 5` - максимум 5 перезавтажень
  - `min_uptime: '60s'` - мінімум 60s між перезавтаженнями
  - `NODE_OPTIONS: --max-old-space-size=512` - обмеження V8 heap

### 2. **Nginx Конфігурація** (`nginx.conf`)
- **Оптимізовано** кешування:
  - Статичні файли тепер кешуються на **1 рік**
  - `index.html` **не кешується** (виключення для SPA)
- **Избегнуто** редиректів циклів:
  - Змінено з `try_files` на `rewrite` з перевіркою файлів
- **Оптимізовано** GZIP:
  - Рівень компресії збільшено до 6
- **Додано** таймаути і ліміти:
  - `client_max_body_size: 10M`
  - `client_body_timeout: 60s`
  - Правильне `Keep-Alive` управління

### 3. **React Компоненти**
- **Carousel.tsx** - закріплено утечку listener'а
  - Раніше не видаляв слухач `reInit` 
  - Тепер видаляє обидва: `reInit` і `select`

### 4. **Автоматичне обслуговування**

#### `maintenance.sh` - Легке очищення (виконується кожні 4 години)
```bash
- Очищення npm кеша
- Видалення старих логів (> 7 днів)
- Ротація великих логів (> 100MB)
- Перевірка статусу PM2
- Плавне перезавантаження додатку
```

#### `deep-clean.sh` - Глибока очистка (виконується щодня о 3 ночі)
```bash
- Зупинення додатку
- Видалення всіх кешів (node_modules, dist)
- Переобудова додатку
- Перезапуск з новим середовищем
```

## 🔧 Установка Cron Job'ів

### Linux/Mac
```bash
crontab -e
```

Додайте ці рядки:
```cron
# Легке очищення кожні 4 години
0 */4 * * * cd /path/to/onlyfa && bash maintenance.sh >> ./logs/maintenance.log 2>&1

# Глибока очистка щодня о 3 ночі
0 3 * * * cd /path/to/onlyfa && bash deep-clean.sh >> ./logs/maintenance.log 2>&1
```

### Windows (з Task Scheduler)
1. Відкрити Task Scheduler
2. Створити нове завдання
3. Встановити тригер: "Щодня о 3:00"
4. Встановити дію: `C:\path\to\bash.exe maintenance.sh`

## 📊 Моніторинг

### Перевірка статусу PM2
```bash
npm run pm2:status
npm run pm2:logs
```

### Перевірка пам'яті
```bash
pm2 monit
```

### Перевірка логів
```bash
tail -f logs/combined.log
tail -f logs/maintenance.log
tail -f logs/err.log
```

## ⚠️ Якщо проблеми повернуться

1. **Перевірте логи**: `tail -f logs/err.log`
2. **Перезавантажте PM2**: `npm run pm2:restart`
3. **Выконайте глибоку очистку**: `bash deep-clean.sh`
4. **Перевірте вільне місце на диску**: `df -h`
5. **Перевірте кількість відкритих файлів**: `lsof | wc -l`

## 📝 Примітки

- Не видаляйте `logs` папку - вона потрібна для моніторингу
- Переконайтеся, що скрипти мають права виконання: `chmod +x *.sh`
- Регулярно переглядайте логи для раннього виявлення проблем
